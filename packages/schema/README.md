# @telekinesis/schema

The shared contract for [Telekinesis](../../README.md): Zod schemas for every
effect, the timesheet, and the sound-profile catalog — plus JSON-Schema exports
for AI tooling.

Every other package depends on this one. If the engine, the CLI, the MCP server
and the UI runtime all agree on these types, a timesheet generated by an LLM is
guaranteed to be playable.

## Usage

```ts
import { parseTimesheet, Timesheet, EFFECT_ACTIONS } from "@telekinesis/schema";

const sheet = parseTimesheet({
  meta: { title: "Pricing flow" },
  timeline: [
    { action: "zoom-in", frameId: "pricing", scale: 1.15 },
    { action: "highlight", frameId: "pro-tier" },
    { action: "cursor-move", destFrameId: "buy" },
    { action: "click", frameId: "buy", soundProfile: "macbook-trackpad" },
  ],
});
// → fully-typed, defaults applied, cross-field rules validated
```

## Exports

| Export | What it is |
| --- | --- |
| `Effect` | Zod discriminated union of all 11 effects |
| `Timesheet` | Zod schema for a full recording score |
| `parseTimesheet` / `safeParseTimesheet` | validate input → typed timesheet |
| `EFFECT_ACTIONS`, `EffectAction`, `EffectOfAction<A>` | action helpers |
| `SOUND_PROFILES`, `SoundProfile` | sound catalog |
| `EasingPattern`, `TransformOrigin`, `CursorCurve` | enums |
| `timesheetJsonSchema`, `effectJsonSchema` | JSON Schema for LLMs/editors |

## Effects

`click`, `type-down`, `drag-and-drop`, `shake`, `zoom-in`, `zoom-out`,
`scroll-up`, `scroll-down`, `cursor-move`, `highlight`, `wait`.

See [`docs/effects.md`](../../docs/effects.md) for every parameter.
